Moderation Commands

m?ban

Bans the given user, with the optional reason being appended

m?channel

Sets the audit channel to whatever channel you pass in here

m?clear

Clears the amount of messages you pass in. The default value is 100

m?editWelcome

Edits the welcome message on the embed send when a member joins the server

m?kick

Kicks the given user

m?info

Gives relevant info about the server.

m?settings

Command to toggle audit and welcome messages in the guild. You need this enabled and the channel to be defined for logging to work

m?unban

Unbans the given user

m?voiceready

Toggles whether to send a startup message in the specified server. It requires an audit channel to be defined to work

m?welcome

Sets the channel to direct welcome messages to the channel you pass in

Music Commands

m?join

Makes the bot join the voice channel you are in

m?leave

Makes the bot leave the voice channel you are in

m?nowplaying

Says the currently playing song

m?pause

Makes the bot pause the currently playing music/video.

m?play

Makes the bot play the video given. It has voice channel join redundancy

m?queue

Sends an embed of the current queue. If it's longer than 20 songs, then it will only display that much and give the current number of songs in queue

m?resume

Makes the bot resume playing the current song

m?skip

Skips the currently playing song

m?stop

Stops all music and clears the queue

Role Commands

m?joinrole

Sets the role to look add to newly joined members, and enables it. Using this with no role will disable it.

m?rolereact

Main command for role reactions

add

Adds a new reaction to listen for automatically adding roles to members

dm

Toggles the dm message settings for role add/remove success.

remove

Removes the reaction from the message, and also stops adding the role to members who choose the reaction

toggle

Sets the roles for the message to toggle on another reaction addition. It automatically removes the old reaction(s)

DM Commands

Commands that require you to DM the bot to trigger

anon send

Sends an anonymous message in the channel specified. Obviously the bot needs to be in the server for it to work.

m?punish

Gives a random "punishment" for the person specified. If no one is specified it defaults to the person who invokes the command
